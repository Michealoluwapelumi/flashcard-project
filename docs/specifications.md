# Phase 4 — Flashcards

## Overview
Implement a complete flashcard study experience for the DevCards app.

## Goals
- Let users review frontend concepts one flashcard at a time.
- Support flipping cards to reveal the answer.
- Allow navigation between previous and next cards.
- Keep the experience responsive and visually polished.

## Functional Requirements
- Display a question on the front of the card.
- Reveal the answer when the card is flipped.
- Provide Previous and Next controls.
- Support filtering flashcards by category.
- Show the current position in the deck.

## UX Notes
- Use a clear visual distinction between question and answer states.
- Make the card interaction feel smooth and lightweight.
- Keep controls easy to reach on both mobile and desktop.

## Acceptance Criteria
- A user can open the flashcards page and study cards in sequence.
- A user can flip a card and view the answer.
- A user can move between cards using navigation controls.
- A user can filter the deck by topic.

# Phase 5 — Categories

## Overview
Organize the learning content into topic-based categories so users can study by theme.

## Goals
- Group flashcards into meaningful categories.
- Make it easy for users to browse available study topics.
- Connect category browsing with the flashcard deck.

## Functional Requirements
- Present category cards with names, descriptions, and card counts.
- Provide a dedicated categories page.
- Allow users to open the flashcard deck for a selected topic.
- Show category-based content from the existing dataset.

## UX Notes
- Use visual cards to make the browsing experience inviting.
- Keep the layout consistent with the home page.
- Make category selection obvious and fast.

## Acceptance Criteria
- A user can visit the categories page and view all available topics.
- A user can identify how many flashcards each category contains.
- A user can navigate from a category to related flashcards.

# Phase 6 — Search & Bookmarks

## Overview
Give users a fast way to find flashcards and save important cards for later review.

## Goals
- Enable quick content discovery through search.
- Support bookmark management for personal study sets.
- Keep saved cards accessible from a dedicated page.

## Functional Requirements
- Search flashcards and categories by keyword.
- Show search results in a dropdown list.
- Bookmark a flashcard from the study deck.
- View bookmarks from a dedicated page.
- Persist bookmarks in local storage.

## UX Notes
- Keep search results concise and helpful.
- Make saved items easy to revisit from the bookmarks page.
- Maintain a polished motion and transition experience.

## Acceptance Criteria
- A user can search for topics and flashcards.
- A user can bookmark a card and later view it in the bookmarks page.
- Bookmarks persist after a refresh.

# Phase 7 — Progress Tracking

## Overview
Help users monitor their learning progress as they study.

## Goals
- Track completed cards.
- Show progress as a percentage.
- Make progress feedback visible during study.

## Functional Requirements
- Mark cards as complete.
- Display a progress bar.
- Show the count of completed cards.
- Persist progress in local storage.

## UX Notes
- Keep the progress indicator lightweight and motivating.
- Pair the metric with the active study flow.

## Acceptance Criteria
- A user can mark a card complete.
- A user can see the progress bar update.
- Progress is preserved after a refresh.

# Phase 8 — Polish

## Overview
Refine the experience to feel complete and production-ready.

## Goals

- Strengthen accessibility and responsiveness.
- Reduce friction in everyday use.
